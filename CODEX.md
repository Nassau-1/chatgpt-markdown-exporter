# Codex Repository Rules

- Treat this repo as a GitHub-first project boundary.
- Keep docs and governance files updated alongside code changes.
- Use normal Markdown links in repo-facing docs.
- Use ISO timestamps for changelog and status updates.
- Add ADRs for material architectural, runtime, or tooling decisions.
- Validate `src/manifest.json` and extension file wiring before closing work.
- Do not commit local conversation exports or browser-specific artifacts.
